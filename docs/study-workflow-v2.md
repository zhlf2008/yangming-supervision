# 学委系统工作流 v2

## 核心规则

1. 人员最终必须落到小组。
2. 每个组织节点都有自己的日程表：大班、班级、小组逐级下钻查看。
3. 大班学委只维护本大班规则；平台管理员可维护全部大班。
4. 保存大班周规则后，系统按学期起止日期自动生成该大班下所有组织节点的日程实例，不再要求手动生成。
5. 每个日程的学习内容按“大班模板 -> 班级覆盖 -> 小组覆盖”继承。
6. 大班共读内容只能由大班节点维护；班级共读内容可由大班模板或班级覆盖；小组共读内容可由大班模板、班级覆盖或小组覆盖。
7. 小组共读可由一个小组自读，也可联合同班其他小组共读。承办组通过岗位需求关联参与组，并指定每个参与组人数。
8. 岗位来自大班岗位表；各级组织安排时选择岗位、填写人数、指定下级组织或同级参与小组。

## 页面分工

- `study-schedule-rules.html`
  - 按大班维护每周共读规则。
  - 大班学委只能改本大班；平台管理员可切换大班。
  - 保存规则后自动生成组织节点日程。

- `study-org-schedule.html`
  - 新的学委主工作台。
  - 支持大班、班级、小组逐级下钻。
  - 展示当前组织的本周日程、继承后的学习内容、内容来源。
  - 可维护岗位表、学习内容、岗位摊派、最终人员落位。

## 数据表映射

- `study_schedule_rules`
  - 增加 `org_id`，表示规则属于哪个大班。
  - 唯一规则为 `semester_id + org_id + weekday`。

- `study_schedule_instances`
  - 继续作为每个组织节点每日日程实例。
  - 生成规则：每个有效日期、每个规则、每个大班/班级/小组组织节点各生成一条实例。

- `study_schedule_content`
  - 新增内容覆盖表。
  - 同一日程实例和组织节点只有一条内容覆盖。
  - 新工作台优先读取此表，旧 `study_schedule_instances.custom_*` 字段作为兼容兜底。

- `study_roles`
  - 新增岗位表。
  - 每个大班维护自己的岗位清单。

- `study_assignment_demands`
  - 增加 `parent_demand_id` 支持逐级摊派。
  - 增加 `role_id` 关联岗位表。
  - 保留 `role_name` 作为历史快照。

- `study_assignment_people`
  - 增加 `org_id`，记录最终落位小组。

## 权限边界

- 规则与岗位：
  - 平台管理员可管理全部。
  - 大班学委可管理本大班。

- 学习内容：
  - 大班学委修改大班节点模板。
  - 班级学委修改班级节点内容。
  - 小组学委修改本小组节点内容。

- 摊派与人员：
  - 当前组织学委可为当前组织日程创建岗位需求。
  - 目标不是小组时，需要继续向下级组织摊派。
  - 目标为小组时，才能选择具体人员落位。
